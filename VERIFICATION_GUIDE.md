# Инструкция по управлению верификацией продавцов

## Как стать администратором

1. **Зарегистрируйтесь на сайте** (если еще не зарегистрированы)
   - Перейдите на http://localhost:3000/auth/register
   - Создайте аккаунт

2. **Сделайте себя администратором через Docker:**

```bash
# Войдите в контейнер backend
docker exec -it diabetic-marketplace-backend sh

# Выполните команду (замените YOUR_EMAIL на ваш email)
npm run make-admin YOUR_EMAIL@example.com

# Выйдите из контейнера
exit
```

Пример:
```bash
docker exec -it diabetic-marketplace-backend sh
npm run make-admin admin@test.com
exit
```

## Как управлять верификациями

### Для администраторов:

1. **Войдите в систему** под аккаунтом администратора

2. **Откройте админ-панель:**
   - Перейдите на http://localhost:3000/admin/verifications
   
3. **Проверьте запросы:**
   - Просмотрите список пользователей, ожидающих верификации
   - Проверьте их документы (ссылки на паспорт, соцсети и т.д.)
   - Прочитайте дополнительную информацию от пользователя

4. **Одобрите или отклоните:**
   - Нажмите "✅ Одобрить" - пользователь станет верифицированным продавцом
   - Нажмите "❌ Отклонить" - укажите причину отклонения

### Для продавцов:

1. **Войдите в свой аккаунт**

2. **Перейдите в профиль:**
   - http://localhost:3000/auth/profile
   - Нажмите кнопку "✅ Стать верифицированным продавцом"

3. **Заполните форму верификации:**
   - Добавьте ссылки на документы (Google Drive, Dropbox и т.д.)
   - Напишите дополнительную информацию о себе
   - Отправьте запрос

4. **Ожидайте проверки:**
   - Статус изменится на "⏳ Проверяется"
   - Обычно проверка занимает 24-48 часов
   - После одобрения получите значок "✅ Верифицирован"

## API Endpoints

### Для пользователей:
- `POST /api/verification/request` - Отправить запрос на верификацию
- `GET /api/verification/status` - Получить свой статус верификации

### Для администраторов:
- `GET /api/verification/pending` - Список ожидающих запросов
- `POST /api/verification/approve/:userId` - Одобрить верификацию
- `POST /api/verification/reject/:userId` - Отклонить верификацию
